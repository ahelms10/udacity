Programming is giving explicit instructions to a computer.

STAGE 1.1

HTML (HyperText Markup Language) docs form the majority of content on the web.

HTML documents are made of HTML elements. When writing HTML, we tell browsers the type of each element by using HTML tags.

Inline vs Block Elements

HTML elements are either inline or block. Block elements form an "invisible box" around the content inside of them.

A browser turns html tags into elements in a tree Browsers know how to translate tags via a document object model (DOM)

Each tag has a name or type that defines what will be created (e.g. paragraph, image, division)

Webpages are comprised of blocks. All shapes have invisible boxes around them. tag > attribute > value

STAGE 1.2

html defines structure and content of page. browser turns html tags into elemennts in a tree; HTML elements can have other elements inside of them

in a browser this structure shows up as a series of nested boxes

programmers use text editors (e.g. Sublime Text, Notepad++, Textpad); the editors provide shortcuts, formatting, intelisense, etc

STAGE 1.3

to define style (i.e. how things look--background, borders, position, colors), use css

browsers know how to interpret css because of the document object model (DOM), which is the aforementioned tree-like structure

each tag has a name or type that defines what will be created (e.g. paragraph, image, division)

the format of css code is:
selector{
attribute: value;
}

an example of the inheritance of cascading style sheets:
national laws
state laws
municipality laws
house rules

css - most specific rule is applied to every elements

specify styles for html tags (e.g p, h1) or classes
